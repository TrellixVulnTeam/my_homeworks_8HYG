Решение: БД
Проще считать статистику, БД за нас решает проблему с одновр. запросами.
Библиотека: стеллажи и книги -- БД Библиотекарь - чувак, кот. определяет, как
распоряжаться книгами -- СУБД (система управления БД)
Определение:  упорядоченное кол-во док-тов, кот. мб обработаны с пом. компьютера
2 вида СУБД (способы управления):
А) реляционные
-- в этот раз
больше всего, чаще используются
Информация хранится в таблицах
Пример: сайт "знаменитые кошки". Есть таблица cats с полями id, name, breed, owner
(создаётся ещё одна таблица с владельцами: имя -- адрес)

Различаются по размеру и назначения: файловые и серверные
Самые популярные:
MySQL
PySQL
SQL Server
Access
SQLite
Язык -- SQL (способ обращения к библиотеке)
structured query language 

Пример команды: SELECT name, breed, FROM cats WHERE id>1
Вернём табличку.
Два основных подмножества операция: 
DDL (data definition language -- создавать и т.д.; будет пользоваться редактором)
4 основных команды: 
CREATE (создать)
LIST (вернуть список таблиц)
DROP (удалить)
ALTER (добавиь поля)
DML (data manipulation language -- управлять)
INSERT (добавлять)
SELECT (получать)
DELETE (удалять)
UPDATE (обновлять)

Б) нереляционные
выглядит как угодно по-другому
Например: mogdgb -- данные json, reges (пары ключ-значение), используется для кэширования

Первая пара: до городов
Вторая пара: база SQLite
